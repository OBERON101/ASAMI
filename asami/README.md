# АСАМИ — Автоматизированная система анализа медицинской инфраструктуры

> Демонстрационный прототип для дипломной работы.  
> Система анализирует медицинскую инфраструктуру Москвы по 12 административным округам,  
> выявляет «белые пятна» и рассчитывает ключевые метрики доступности МО.

---

## Быстрый старт

### 1. Установите Python 3.11+

Скачайте с официального сайта: **https://www.python.org/downloads/**

Проверьте версию в терминале:
```bash
python --version
# Python 3.11.x
```

### 2. Создайте виртуальное окружение (рекомендуется)

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python -m venv venv
source venv/bin/activate
```

### 3. Установите зависимости

```bash
cd asami
pip install -r requirements.txt
```

> ⏱ Установка занимает 2–5 минут в зависимости от скорости интернета.

### 4. Запустите приложение

```bash
streamlit run app.py
```

Браузер откроется автоматически по адресу **http://localhost:8501**

## Описание вкладок

### 📊 Мониторинг

Общая картина медицинской инфраструктуры города:

- **4 KPI-карточки**: всего МО, средняя плотность, округа с высокой доступностью, белые пятна
- **Интерактивная choropleth-карта** — округа залиты цветом по плотности МО (зелёный = норма, красный = дефицит). Нажмите на округ для просмотра детальной информации
- **Рейтинговая таблица** с цветовым статусом каждого округа
- **Горизонтальный bar chart** — визуальное сравнение плотности МО

### 🔍 Фильтрация

Анализ конкретного типа медицинских организаций:

- Radio-кнопки для выбора типа: Все / Больницы / Поликлиники / Амбулатории / Специализированные
- Карта автоматически обновляется и показывает только МО выбранного типа
- Таблица со списком конкретных МО (название, адрес, источник данных, оценка достоверности)
- Pie chart распределения МО по типам

### ⚠️ Белые пятна

Выявление зон с дефицитом медицинской помощи:

- Ползунок для настройки порогового значения density
- Карта: красные округа = ниже порога, серые = норма
- Раскрывающийся список проблемных округов с рекомендациями
- Сводная таблица с расчётом дефицита (сколько МО нужно добавить)
- **Кнопка экспорта отчёта в CSV**

## База данных (опционально)

Приложение **работает без БД** (используются демо-данные).

Для подключения к реальной PostgreSQL/PostGIS:

1. Установите PostgreSQL 14+ и расширение PostGIS
2. Создайте БД и пользователя:
   ```sql
   CREATE DATABASE asami_db;
   CREATE USER asami_user WITH PASSWORD 'asami_pass';
   GRANT ALL ON DATABASE asami_db TO asami_user;
   ```
3. Настройте параметры в `config.py` или создайте файл `.env`:
   ```
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=asami_db
   DB_USER=asami_user
   DB_PASSWORD=asami_pass
   ```

## Решение проблем

**`ModuleNotFoundError: No module named 'streamlit'`**  
→ Убедитесь что виртуальное окружение активировано и выполнен `pip install -r requirements.txt`

**`Port 8501 is already in use`**  
→ Запустите на другом порту: `streamlit run app.py --server.port 8502`

**Карта не отображается**  
→ Проверьте установку пакетов: `pip install folium streamlit-folium`

**Медленный первый запуск**  
→ Это нормально — Streamlit компилирует компоненты. Повторный запуск быстрее.
